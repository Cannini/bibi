Modo apresentação

Arquitetura do Stop Online

Fluxo usado na defesa: API FastAPI centraliza regras, Redis compartilha estado, RabbitMQ audita eventos e WebSocket atualiza os jogadores em tempo real.

Redis / cache

As salas ficam em chaves stop:room:{id}. A API tenta cache primeiro, registra hit/miss/erro e usa origem em memória como fallback quando configurado.

RabbitMQ

Eventos room.* são publicados em tópico e consumidos pela fila de auditoria stop.events.audit.

WebSocket

Cada mudança relevante dispara broadcast para os sockets conectados e mantém lobby, rodada, votação e pódio sincronizados.
